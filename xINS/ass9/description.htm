Secure Chatt

Av Carina Nerén, 2004

 

 

 


Inledning:

 

Secure Chatt är en chatt som utövar start säkerhet genom att både

signera och kryptera chattmeddelanden.

 

Hur man startar:

 

Spelet startas med att man i ett kommandofönster skriver:

 

java –jar Client.jar <host> <port>

 

där host är adressen till servern som chatten ska koppla upp sig mot,

och port är den port som används. Om man inte definierar någon host

eller port sätts dessa värden till defaultvärderna 127.0.0.1 respektive 2000.

 

Beskrivning:

 

När användaren startat upp sin chatclient får denne upp ett chatfönster

där det i textarean står ett meddelande om att denne kan skriva in ett

användarnamn som sedan blir användarens identifierare.

 

Bild 1. Ett chattfönster har öppnats.
 

 

 

 

 

 


Meddelandet från servern byter färg för att indikera att användaren har

skickat sitt namn, därefter skickas ett nytt meddelande från servern som

talar om att användaren har anslutit sig till chatten.

 

Bild 2. Två användare har anslutit sig till chatten.
 

 

 

 

 

 


Därefter kan användaren börja chatta och före varje meddelande skrivs

namnet som användaren valt att identifiera sig med ut.

 

Bild 3. Två användare har börjat chatta med varandra.
 

 

 

 

 

 


Varje gång en användare har skickat ett meddelande signeras och

krypteras det innan det skickas vidare till servern som i sin tur

skickar det till resterande uppkopplade chatklienter. Meddelandet

avkrypteras och veriferas av de mottagande klienterna och om

meddelandet är giltigt skrivs det ut annars ges ett felmeddelande

till klienten om att meddelandet är korrupt.

 

Bild4. Gränssnittet på servern som visar att två chattklienter är uppkopplade.
